Flipkart Wishlist Manager App Architecture Diagram

1. Add Item Form (UI Input)
↓ (Form Submit Data)
2. Storage (JavaScript Array / localStorage)
↓ (Re-render Event)
3. Display Wishlist (UI List)
↑ ↓ (Edit / Delete Actions Trigger Update)
4. Mutate Array & Refresh UI

Data Flow Explanation:

  • Add Item Form: Captures product title, price, and image URL. On submission, data is packaged into an object.
  • Storage: The object is pushed to a global JavaScript array (and optionally serialized to localStorage to persist data).
  • Display Wishlist: A rendering function loops through the storage array and dynamically generates HTML elements to show the items.
  • Edit/Delete Actions: Clicking 'Delete' splices the element from the array using its index. Clicking 'Edit' updates fields. Both actions immediately trigger the rendering function to keep the UI in sync.
